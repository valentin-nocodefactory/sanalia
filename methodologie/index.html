Méthodologie de la carte des nuisibles en France

Ce document décrit en toute transparence les sources, les méthodes de collecte, les calculs et les limites des données présentées sur la carte interactive Sanalia. Il s'adresse aux journalistes, chercheurs et collectivités qui souhaitent comprendre, citer ou contester nos chiffres.

En bref

La carte interactive Sanalia agrège 5 sources publiques officielles et les données opérationnelles anonymisées de Sanalia pour cartographier la présence des principaux nuisibles sur le territoire français : rats, moustique tigre, frelon asiatique, punaises de lit et ambroisie.

1.6 Données Sanalia (en complément)

Lorsque les données publiques sont absentes ou peu denses sur une commune, la carte peut intégrer en complément les interventions Sanalia anonymisées : présence/absence binaire au niveau de la commune, sans détail client, sans adresse précise. Aucune donnée personnelle, identifiant ou adresse n'est exposée.

Cette intégration est :

  • explicitement marquée sur la carte (source : sanalia_internal),
  • soumise à un seuil minimal de 3 interventions sur la commune avant publication, pour éviter la ré-identification,
  • documentée dans le détail au cas par cas sur demande presse motivée.

2. Méthodologie de collecte

2.1 Pipeline technique

  1. Extraction quotidienne automatisée via le script fetch_nuisibles_data.py (open source, disponible sur demande).
  2. Normalisation des signalements vers un schéma unique (Signalement : nuisible, source, date, lat/lng, code INSEE, commune, département, intensité).
  3. Enrichissement géographique : géocodage inverse via api-adresse.data.gouv.fr (BAN officielle) pour récupérer le code INSEE quand la source ne le fournit pas.
  4. Agrégation par commune INSEE : somme des signalements, ventilation par nuisible, par source.
  5. Normalisation per capita : taux pour 1 000 habitants à partir de la population légale INSEE millésime en cours.
  6. Export GeoJSON sous licence ODbL pour ré-utilisation par tiers (médias, chercheurs).

2.2 Périmètre temporel

  • Signalements ponctuels : 12 derniers mois glissants (date de référence : J-3 mois pour Paris).
  • Données régionales punaises de lit : 2017–2022 (figées tant que l'ANSES n'a pas publié de mise à jour).
  • Communes colonisées moustique tigre : dernière liste annuelle ANSES (millésime 2026).

4. Limites et biais — à lire avant toute citation

Nous tenons à expliciter les biais inhérents aux sources, conformément aux bonnes pratiques de data journalisme.

4.1 Biais déclaratif

Les données Dans Ma Rue, signalement-moustique, signalement-ambroisie et signalement frelon reposent sur des contributions citoyennes volontaires. Une commune avec beaucoup de signalements n'est pas nécessairement plus infestée — elle est aussi plus mobilisée numériquement. Les écarts territoriaux peuvent refléter une sensibilisation locale plus que la prévalence biologique réelle.

4.2 Couverture inégale

  • Paris dispose d'une plateforme depuis 2012 ; la majorité des communes françaises n'ont pas d'équivalent → sur-représentation parisienne mécanique.
  • L'ANSES couvre l'ensemble du territoire pour le moustique tigre, mais avec un délai de validation pouvant atteindre plusieurs mois.

4.3 Sources hétérogènes

Les sources ne sont pas homogènes en méthodologie : un signalement Dans Ma Rue ≠ une intervention SMASH ≠ une commune classée colonisée par l'ANSES. Nous comptons des événements, pas des populations biologiques. Les comparaisons inter-sources sont donc à manier avec prudence.

4.4 Punaises de lit — granularité régionale uniquement

La granularité régionale est imposée par la source ANSES. Toute lecture commune par commune est trompeuse sur ce nuisible spécifique. Les chiffres « 100 000 signalements à Paris en 2023 » largement repris dans la presse proviennent d'agrégats municipaux distincts et ne sont pas comparables aux 11 % nationaux de l'ANSES (méthodes d'estimation différentes).

4.5 Sanalia n'est pas neutre

Sanalia est une entreprise de lutte contre les nuisibles. Nous avons intérêt à ce que le sujet soit visible. Pour limiter ce biais :

  • nous publions le code source du pipeline (vérifiable),
  • nous séparons clairement les données publiques (en accès libre) de nos propres interventions (marquées comme telles),
  • nous nous engageons à ne jamais modifier les données publiques en aval.

5. Mise à jour et reproductibilité

  • Cadence de mise à jour : mensuelle (1er du mois).
  • Code source du pipeline : disponible sur demande à data@sanalia.fr (Python, MIT licence).
  • GeoJSON brut téléchargeable : /data/nuisibles_france.geojson
  • Stats régionales punaises (JSON) : /data/punaises_lit_regional.json

Toute personne souhaitant reproduire l'analyse peut le faire intégralement à partir des sources publiques citées, sans accès privilégié.

6. Pour citer cette carte

Sanalia (année). Carte des nuisibles en France. Observatoire Sanalia. Disponible sur sanalia.fr/map-interactive. Données : Paris Open Data, ANSES, MNHN/INPN, Atlasanté/FREDON, INSEE.

Mention recommandée pour les médias : « D'après l'Observatoire Sanalia, croisant les données publiques de l'ANSES, du MNHN et des collectivités ».

7. Demandes presse — angles déjà disponibles

Sur demande motivée à presse@sanalia.fr, nous fournissons gratuitement :

  • les données brutes par département ou par région (CSV / GeoJSON),
  • des graphiques prêts à publier (PNG haute définition, SVG modifiable),
  • une note d'analyse de 1–2 pages pour contextualiser un chiffre dans son territoire,
  • l'embed iframe de la carte interactive avec votre angle pré-filtré (par exemple : carte zoomée sur la Métropole de Lyon, ou ne montrant que les rats),
  • une interview avec notre responsable opérationnel ou un expert du secteur (entomologiste partenaire).

Délai de réponse standard : < 24 h ouvrées.
